Arquitectura Cliente/Servidor

La arquitectura cliente-servidor constituye el modelo fundamental que permite el funcionamiento dinámico de la web. Gracias a esta estructura, las aplicaciones no se limitan a mostrar documentos estáticos, sino que ofrecen servicios interactivos y personalizados. En este modelo, el lado del cliente —representado por el navegador— se encarga de interactuar con el usuario y solicitar recursos, mientras que el servidor procesa la lógica del sistema, administra las bases de datos y responde a múltiples solicitudes de manera simultánea.

Su importancia radica en la centralización de la información y en la posibilidad de escalar el servicio para atender a numerosos usuarios al mismo tiempo. Mediante protocolos de comunicación establecidos, el servidor entrega información procesada en tiempo real, como resultados de búsqueda, correos electrónicos o contenido multimedia, garantizando eficiencia y organización en la gestión de datos.

... ofreciendo eficiencia y organización en la gestión de datos (Luján, 2002; Infranetworking, 2019).

Es importante destacar que, en ciertos contextos, un cliente puede asumir funciones de servidor al almacenar datos localmente para evitar consultas repetitivas, optimizando así el rendimiento del sistema. Este comportamiento demuestra cómo las especificaciones técnicas pueden ajustarse para mejorar la eficiencia en la comunicación entre cliente y servidor.

Diseño Centrado en el Usuario

El diseño centrado en el usuario es un enfoque metodológico que prioriza las necesidades, capacidades y expectativas de las personas que interactuarán con el sistema. Este modelo busca desarrollar interfaces intuitivas, accesibles y eficientes, garantizando que la tecnología se adapte al usuario y no al contrario.

Para lograrlo, se realizan análisis previos del contexto, pruebas de usabilidad y procesos iterativos de mejora. De esta manera, el sistema final no solo cumple con requisitos técnicos, sino que también ofrece una experiencia de uso satisfactoria.

  • Usabilidad
  • Accesibilidad
  • Experiencia de usuario
  • Evaluación continua

... garantizando que la tecnología se adapte al usuario y no al contrario (Granollers, 2012).

Importancia de la Investigación

La investigación en el territorio es un paso determinante para el análisis de requisitos y la definición de especificaciones técnicas en programación web. Este proceso permite identificar las condiciones reales del entorno, como la capacidad de la red, el tipo de hardware disponible y las necesidades específicas de la organización.

Comprender el contexto facilita la elección adecuada de la arquitectura tecnológica, ya sea en dos capas, tres capas o múltiples capas, optimizando el flujo de información y garantizando la viabilidad del proyecto. Además, permite anticipar aspectos relacionados con la seguridad, el presupuesto y la sostenibilidad del sistema.
